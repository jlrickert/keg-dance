# System: Entity selection guide - dance

A decision tree for choosing the right [entity type](../6) when creating a
new note in the dance KEG. This KEG uses 12 entity types organized into
five groups. When in doubt, start with [draft](../17) and promote later.

## Decision tree

### Is it about how this KEG works?

- Defining a note type -> [entity](../6)
- KEG conventions, processes, or rules -> [system](../7)

### Is it capturing dance knowledge?

- A reusable dance concept, principle, or technique -> [concept](../8)
- A stable lookup note (timing charts, style comparisons) -> [reference](../9)
- A step-by-step how-to or practice drill -> [guide](../10)
- A summary of a video, blog post, or workshop -> [article](../11)

### Is it about a specific dance move?

- An individual move, pattern, or combination -> [move](../16)

### Is it about people, places, or events?

- An instructor, partner, or notable dancer -> [person](../12)
- A venue, studio, or practice space -> [place](../13)
- A social, workshop, festival, or class -> [event](../14)
- A dance company, school, or community group -> [organization](../15)

### Not sure yet?

- Work-in-progress content not yet finalized -> [draft](../17)

## Key distinctions

| Confusion | Resolution |
|-----------|------------|
| concept vs move | A concept explains a principle (e.g. lead-follow connection, frame); a move is a specific named pattern (e.g. cross-body lead, swingout) |
| concept vs reference | A concept explains *what* something is; a reference is a lookup table for *how* to use it |
| guide vs reference | A guide walks through steps; a reference is a flat lookup |
| guide vs article | A guide is original instruction; an article summarises external content |
| move vs guide | A move documents one pattern; a guide teaches a workflow or drill |
| place vs organization | A place is a physical location; an organization is a group that may use multiple places |
| event vs place | An event is a scheduled occurrence; a place is where events happen |
| event vs organization | An event is a single occurrence; an organization runs multiple events |
| person vs organization | A person is an individual; an organization is a group |
| draft vs any | Draft is temporary -- promote to the real entity when ready |
